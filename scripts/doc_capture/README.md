# Documentation Capture Harness

This proof-of-concept harness renders deterministic MagScope GUI assets for documentation work.
It is intended for guide generation, not GUI regression testing.

## Run

From the repository root on Windows:

```powershell
.\.venv\Scripts\python.exe .\scripts\doc_capture\capture_gui.py --visible
```

By default, assets are written to the formal guide asset location:

```text
assets/doc_capture/
```

Use a scratch output directory while developing:

```powershell
.\.venv\Scripts\python.exe .\scripts\doc_capture\capture_gui.py --visible --output .\.codex-tmp\doc_capture
```

Use `--dry-run` to build the capture scene and list the planned outputs without writing files.

Use the default offscreen mode only for quick dry runs or headless environments:

```powershell
.\.venv\Scripts\python.exe .\scripts\doc_capture\capture_gui.py --dry-run
```

The `--visible` mode shows a real Qt window through the native desktop Qt backend, then captures
each widget through Qt's widget-grab path. That path is preferred for guide assets because it
matches normal user-facing text rendering more closely than Qt's offscreen backend without
depending on desktop window stacking.
Embedded Matplotlib canvases are composited from direct figure renders during visible capture so
the guide keeps readable native Qt text and reliable plot regions.

## Assemble GIFs From Frames

The reusable GIF assembler turns any ordered PNG frame folder into an animated GIF. It works for
full-window captures, dock/widget captures, or any other component screenshots as long as every
frame in a single sequence has the same dimensions.

Recommended frame naming:

```text
frame-01.png
frame-02.png
frame-03.png
```

The default frame pattern is `frame-*.png`. Frames are sorted by filename using natural numeric
order, so zero-padded names are preferred and easiest to review.

Example using an existing capture sequence:

```powershell
.\.venv\Scripts\python.exe .\scripts\doc_capture\assemble_gif.py `
  .\assets\doc_capture\gif_frames\workflows\bead-selection-workflow `
  .\assets\doc_capture\gifs\workflows\bead-selection-workflow.gif
```

Use `--duration-ms` for stable timing across every frame:

```powershell
.\.venv\Scripts\python.exe .\scripts\doc_capture\assemble_gif.py `
  .\assets\doc_capture\gif_frames\workflows\bead-selection-workflow `
  .\assets\doc_capture\gifs\workflows\bead-selection-workflow.gif `
  --duration-ms 350
```

Use `--resize`, `--max-width`, `--max-height`, and `--optimize` for doc-friendly output size:

```powershell
.\.venv\Scripts\python.exe .\scripts\doc_capture\assemble_gif.py `
  .\assets\doc_capture\gif_frames\workflows\bead-selection-workflow `
  .\assets\doc_capture\gifs\workflows\bead-selection-workflow-small.gif `
  --max-width 640 `
  --optimize
```

The assembler reads the PNG frames and writes the GIF. It does not delete, move, or rewrite the
source PNG frames, so the frame sequence remains available locally for review and regeneration.
The default `assets/doc_capture/gif_frames/` folder is ignored by Git; commit the final GIF, not
the intermediate frame PNGs.

## Cursor-Overlay Workflow GIFs

The bead-selection workflow is generated by sending real Qt mouse events to the documentation
`VideoViewer`. The capture harness records the same scene coordinates used for those clicks and
drags, captures source PNG frames after each important GUI state change, and then draws a synthetic
cursor plus click/drag rings into the stored frame sequence.

This keeps the workflow behavior tied to the real GUI event path while making the cursor visual
stable and readable in the final GIF.

## What It Captures

The script builds a real Qt window from MagScope UI classes without launching the full multi-process
application. It uses the default simulated bead camera to render a live-camera image and overlays
representative bead ROIs.

Current outputs include:

- full main window screenshot
- live camera dock screenshot
- Live Plots dock screenshots for the startup no-data state and a populated track-data state
- full Run, Analysis, Locking, and Motors tab screenshots
- individual panel screenshots for status, acquisition, camera settings, scripting, plot settings,
  histogram, radial profile, XY-Lock, Z-Lock, and hardware managers
- workflow screenshots for the Analysis tab with live plot data, auto bead selection, accepted
  auto-selected ROIs, saving enabled, and the save-folder picker
- a bead-selection workflow frame sequence with a synthetic cursor overlay
- an animated GIF of bead selection when Pillow is installed
- Z-LUT setup, generated-Z-LUT review, simulated focus motor panel, and simulated focus motor
  plot screenshots
- search box screenshot with the completion results dropdown
- Preferences dialog screenshot

If Pillow is not installed, the PNG frame sequence is still written locally and can be converted
later.
The capture harness uses the same GIF assembler internally when GIF output is enabled.

See `asset_map.md` for the first beginner-guide outline and the mapping between guide sections
and generated assets.

## Current Boundary

This is a documentation asset generator. The bead-selection GIF uses real Qt events inside the
in-process documentation scene, but the harness does not yet drive an already-running MagScope
process with OS-level mouse and keyboard events, and it does not verify GUI behavior as a test
suite.
